Задание на учебную практику

Проект: "Разработка модели компетенций специалиста ИТ-сферы"

Цель практики: формирование у обучающихся профессиональных компетенций в области анализа требований к ИТ-специалистам, проектирования баз данных и применения OLAP-аналитики для построения модели компетенций.
Задачи:
Провести анализ ФГОС ВО, профессиональных стандартов и вакансий для выявления ключевых компетенций ИТ-специалистов.
Разработать структуру базы данных для хранения и обработки данных о компетенциях.
Применить OLAP-аналитику (ClickHouse) для визуализации и анализа данных.

Выделить обобщенные трудовые функции (ОТФ) и трудовые действия (ТД).

OLAP-аналитика в ClickHouse (ПК-2.1, ПК-1.3)

Активности:
Настроить ClickHouse.
Загрузить данные из PostgreSQL в ClickHouse (например, через clickhouse-client или Altinity – инструменты миграции).
Создать таблицу в ClickHouse для OLAP-анализа.
Создать куб данных для анализа:
CREATE TABLE competencies_olap (
    competency String,  -- Название компетенции (SQL, Python и т.д.)
    source String,      -- Источник (ФГОС, Профстандарт, Вакансия)
    count UInt32        -- Количество упоминаний
) ENGINE = MergeTree()
ORDER BY (competency, source);

Заполнение данными:
INSERT INTO competencies_olap VALUES
    ('SQL', 'ФГОС', 15),
    ('SQL', 'Профстандарт', 40),
    ('SQL', 'Вакансия', 95),
    ('Python', 'ФГОС', 10),
    ('Python', 'Вакансия', 80),
    ('Docker', 'Вакансия', 70),
    ('ClickHouse', 'Профстандарт', 5);

Выполнить запросы с GROUP BY CUBE:
SELECT competency, source, SUM(count) 
FROM competencies_olap 
GROUP BY CUBE(competency, source);  

Визуализировать результаты в Grafana или Metabase.

Способы представления результатов:

А) Примеры OLAP-запросов с выводом и интерпретацией результата:
Базовый запрос. Группировка по компетенциям – узнать суммарное количество упоминаний каждой компетенции (все данные).
SELECT 
    competency, 
    SUM(count) AS total_mentions
FROM competencies_olap
GROUP BY competency
ORDER BY total_mentions DESC;

Базовый запрос. Анализ по источникам – сравнить, как часто навыки (выборка) встречаются в разных источниках.
SELECT 
    competency, 
    source, 
    SUM(count) AS mentions
FROM competencies_olap
GROUP BY competency, source
ORDER BY competency, mentions DESC;

Базовый запрос: TOP-N компетенций в вакансиях – вывести 5 самых востребованных навыков в вакансиях.
SELECT 
    competency, 
    SUM(count) AS total
FROM competencies_olap
WHERE source = 'Вакансия'
GROUP BY competency
ORDER BY total DESC
LIMIT 5;

Базовый запрос: компетенции, которых нет в ФГОС – найти навыки, популярные в вакансиях, но отсутствующие в образовательных стандартах.
SELECT DISTINCT competency
FROM competencies_olap
WHERE source = 'Вакансия'
AND competency NOT IN (
    SELECT competency 
    FROM competencies_olap 
    WHERE source = 'ФГОС'
);

Расширенный запрос. Агрегация с CUBE – получить сводную статистику по всем возможным группировкам.
SELECT 
    competency, 
    source, 
    SUM(count) AS total
FROM competencies_olap
GROUP BY CUBE(competency, source)
ORDER BY competency, source;

Расширенный запрос. Агрегация с ROLLUP (иерархическая агрегация) – посчитать вложенные итоги (например, общее по источнику → детализация по компетенциям).
SELECT 
    source, 
    competency, 
    SUM(count) AS total
FROM competencies_olap
GROUP BY ROLLUP(source, competency)
ORDER BY source, competency;

Б) Дашборд с вкладками (визуализация в Grafana). Например, «частотность навыков» (столбчатая диаграмма), «сравнение источников» (pie-чарт), «динамика спроса» (временной график).

В) Рекомендации на основе анализа (например, навык N встречается в 70% вакансий, но отсутствует в ФГОС – требуется актуализация образовательной программы).

Г) Аналитический отчет (топ-20 востребованных навыков, топ-10 востребованных технологий).

Д) Видеодемонстрация (3-5 минут) – запись экрана: парсинг данных → работа с ClickHouse → дашборд.
