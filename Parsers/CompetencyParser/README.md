# Система анализа компетенций ИТ-специалистов

## Описание проекта

Данный проект реализует первый пункт задания на учебную практику - **"Анализ нормативно-регламентирующих документов и вакансий (ПК-1.1, ПК-2.2)"**.

Система предназначена для:
- Парсинга данных ФГОС ВО по ИТ-направлениям
- Анализа профессиональных стандартов ИТ-сферы
- Сбора и анализа требований из вакансий
- Выявления ключевых компетенций и их частотности

## Структура проекта

```
CompetencyParser/
├── Models/                     # Модели данных
│   ├── Competency.cs          # Основная модель компетенции
│   ├── FgosData.cs           # Модель данных ФГОС
│   ├── ProfessionalStandardData.cs # Модель профстандартов
│   └── VacancyData.cs        # Модель вакансий
├── Parsers/                   # Парсеры данных
│   ├── IParser.cs            # Базовый интерфейс
│   ├── FgosParser.cs         # Парсер ФГОС
│   ├── ProfessionalStandardParser.cs # Парсер профстандартов
│   └── VacancyParser.cs      # Парсер вакансий
├── Services/                  # Сервисы
│   ├── CompetencyAnalyzer.cs # Анализатор компетенций
│   └── TzReader.cs           # Читатель технического задания
├── Results/                   # Результаты парсинга
└── Program.cs                # Главный файл

```

## Технологии

- **C# .NET 8.0** - основной язык разработки
- **HtmlAgilityPack** - для парсинга HTML-страниц
- **Newtonsoft.Json** - для работы с JSON
- **CsvHelper** - для экспорта в CSV
- **HeadHunter API** - для получения данных о вакансиях

## Основные функции

### 1. Анализ технического задания
- Чтение файла TZ.txt
- Извлечение ключевых требований
- Создание краткого анализа

### 2. Парсинг ФГОС ВО
- Анализ направлений 02.04.03, 09.04.01, 09.04.04
- Извлечение кодов и описаний компетенций
- Формирование индикаторов достижения

### 3. Парсинг профессиональных стандартов
- Анализ стандартов: "Программист", "Системный аналитик", "Архитектор ПО"
- Выделение ОТФ и трудовых действий
- Извлечение требуемых навыков и знаний

### 4. Парсинг вакансий
- Интеграция с HeadHunter API
- Сбор требований к навыкам
- Определение частотности упоминаний технологий

### 5. Анализ и агрегация
- Объединение данных из всех источников
- Подсчет частотности компетенций
- Категоризация навыков

## Результаты работы

Программа создает следующие файлы в папке `Results/`:

1. **tz_analysis.txt** - анализ технического задания
2. **fgos_data.json** - данные ФГОС в JSON формате
3. **prof_standards_data.json** - данные профстандартов
4. **vacancy_data.json** - данные вакансий
5. **competencies_analysis.csv** - сводная таблица компетенций
6. **analysis_report.txt** - аналитический отчет

## Установка и запуск

1. Убедитесь, что установлен .NET 8.0 SDK
2. Перейдите в папку проекта:
   ```bash
   cd "c:\Users\Home\Desktop\UniversityPractice\GrabberInformationApi\Parsers\CompetencyParser"
   ```
3. Восстановите пакеты:
   ```bash
   dotnet restore
   ```
4. Запустите проект:
   ```bash
   dotnet run
   ```

## Особенности реализации

- **Модульная архитектура** - каждый парсер реализует единый интерфейс
- **Асинхронные операции** - все операции ввода-вывода выполняются асинхронно
- **Обработка ошибок** - предусмотрена обработка ошибок сети и API
- **Тестовые данные** - в случае недоступности внешних источников используются mock-данные
- **Кодировка UTF-8** - корректная работа с русским текстом
